# 楼中楼“拖尾”文件实现指南



目标：在输出合并去重数据后，对其中的楼中楼评论的互相评论逻辑进行优化和整理：

如主楼评论A下面有BCDEFGHI八条评论，检查他们的rpid和parent数据；

发现BCDE的parent和A的rpid一致，FG的parent和B的rpid一致，H的parent和D的rpid一致，I的parent和G的rpid一致

则最终的连接顺序是：

```
A
├── B
│   ├── F
│   └── G
│       └── I
├── C
├── D
│   └── H
└── E
```



而在实际生成的图中，“A”应当替换为“{用户名称}：{评论内容}”的格式



特别的，只对楼中楼回复超过5条的评论进行生成拖尾文件的操作
